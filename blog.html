does it matter to use relative sizes for styling?

before answering the question i would like you to think of building a hospital. Now forget all the details and focus on one question: "Would you like your hospital to have ramps and elevators?"

the answer to both questions would be yes. Ofcourse the second one is somewhat obvious- who would build a hospital that is inaccessible to the patients? Right! The thing is the first question has a very similar logic as to why webpages should be styled using relative sizes.

setting the sizes for visual components such as spacing, font size, and the like is simply not considering the end-users. When a person creates a websites, the targets of the website should be all of the population and among that population are people who are in their 60s with visual impairments who may not be able to see the fixed pixel size that the site used.

while the default medium font size is not usually changed by the end-users- and even less frequently by 60 year-olds. However it is possible to do so in all browsers. When the users change that font, if the styling of the website uses fixed sizes like pixels to specify sizes, then there would be no effect meaning the user is not going to enjoy browsing the site due to its uncomfortable sizing. On the other hand if the site uses relative sizes like em, rem, and % which depend on the default size of the browser to style the site, it means that whenever the default font of the browser changes along with it changes the content on the website.

to formally address the topic of the article, yes it matters a lot to use relative sizes for styling
